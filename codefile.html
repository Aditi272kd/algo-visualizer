Bubble Sort
Repeatedly swaps adjacent out-of-order pairs. Simple but O(n²) average. Best case O(n) with early termination when no swaps occur.
Best: O(n)Worst: O(n²)Stable
Selection Sort
Finds minimum from unsorted portion each pass. Makes exactly n-1 swaps — useful when write operations are costly. Always O(n²).
Always: O(n²)Space: O(1)
Insertion Sort
Excellent for nearly-sorted data and small arrays. O(n) best case. Used as a base case in TimSort and introsort hybrid algorithms.
Best: O(n)Worst: O(n²)Stable
Merge Sort
Divide-and-conquer; guaranteed O(n log n) in all cases. Requires O(n) extra space. Preferred for linked lists and external sorting.
All: O(n log n)Space: O(n)Stable
Quick Sort
Average O(n log n) with excellent cache performance and in-place partitioning. Worst case O(n²) with bad pivots; mitigated by randomization.
Avg: O(n log n)Worst: O(n²)Stack: O(log n)
Counting Sort
Non-comparison sort. Counts element frequencies then reconstructs. O(n+k) where k = value range. Only for non-negative integers with bounded range.
Linear: O(n+k)Space: O(k)Stable
Radix Sort
Sorts digit by digit using a stable subroutine (LSD/MSD). O(nk) where k = number of digits. Beats comparison sorts for fixed-width integers.
O(n·k)Space: O(n+b)Stable
