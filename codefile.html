Bubble Sort
Repeatedly swaps adjacent elements that are out of order. The largest elements "bubble up" to the end each pass. Simple to understand but inefficient for large arrays.
Best: O(n) Worst: O(n²) Stable
Selection Sort
Finds the minimum element from the unsorted portion and swaps it to the correct position. Makes exactly n-1 swaps, making it useful when swaps are costly.
Always: O(n²) Space: O(1)
Insertion Sort
Builds a sorted subarray one element at a time. Excellent for nearly-sorted data and small arrays. Used as the base case in hybrid sorting algorithms.
Best: O(n) Worst: O(n²) Stable
Merge Sort
Divide-and-conquer: recursively splits the array in half, sorts each half, and merges them. Guaranteed O(n log n) in all cases. Requires O(n) extra space.
All cases: O(n log n) Space: O(n) Stable
Quick Sort
Picks a pivot and partitions the array around it. Average O(n log n) with excellent cache performance. Worst case O(n²) occurs with bad pivot selection.
Avg: O(n log n) Worst: O(n²) Space: O(log n)
